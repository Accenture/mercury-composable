### Part A — flip the ElasticQueue default to `file` (`6e21fcd1`)
`ElasticQueue.create()` now defaults `elastic.queue.store` to `file` (was `bdb`); `bdb` is the explicit
legacy fallback, and any unrecognized value falls through to `file`. Dispatch still derives from the store
(file ⇒ off-loop vthread, bdb ⇒ inline loop), so vthread+bdb stays unreachable. Updated `ElasticQueue` +
`BdbElasticStore` doc comments. **Verified: platform-core 381 green with the flipped default (no store flag)**
— nothing depended on the bdb default (only the two gated benchmarks set the store explicitly). Rewrote the
canary field notes for the inverted framing (file = default; fallback = `elastic.queue.store=bdb`; removing
the setting no longer selects bdb).

### Part B — new `benchmark-reporter` module (`e4c39aee`)
`benchmark/benchmark-reporter` — a self-contained, single-JVM end-to-end performance harness (kept separate
from `benchmark-client`, which is left for a future sprint). Uses the Mercury app idiom (`@MainApplication` +
`AutoStart`, spring-boot-maven-plugin repackage → executable jar; headless, no REST). Drives an echo
`benchmark.worker` two ways:
- **RPC** — closed-loop `request()`, in-flight bounded by concurrency (default 1 ⇒ pure round-trip, no
  ElasticQueue).
- **Callback** — open-loop `asyncRequest()` with bounded in-flight (default 2000 ≫ 20 memory buffer) ⇒
  floods the worker ⇒ **exercises the ElasticQueue spill** (the only path that does, as Eric noted).

Captures per-op end-to-end latency into a preallocated array (atomic index, race-free, no hot-path
allocation), computes exact percentiles + mean/stddev + log-spaced histogram (`Stats`), and renders a
**self-contained HTML report** (`HtmlReport`): inline CSS + inline SVG histogram + percentile "nines" plot +
stats table + environment metadata (records active store + dispatch mode so file vs bdb can be A/B'd). All
params are `-Dbench.*` system properties; runs anywhere a JRE does (field env / pipeline). README documents
usage (incl. the `-D`-before-`-jar` gotcha).

**Verified:** compiles; executable jar runs; 5k smoke + 200k full run both exit 0 with `store=file`,
`dispatch=vthread`, `FileElasticStore` engaged. Results tell the intended story — RPC mean 0.047ms
(p99 0.076) vs Callback mean 65.7ms (2000-deep queue at one worker ≈ Little's law). HTML confirmed
self-contained (only the SVG xmlns URL; no link/script/CDN); SVG coords on-canvas, bins land where the stats
say. Sample reports are gitignored (module `.gitignore` + root `benchmark-report*.html` guard).

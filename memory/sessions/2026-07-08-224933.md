## Feature: graph.task — invoke a composable function from a graph node
Eric's design intent (CONFIRMED, 5 decisions): a Task node with `skill=graph.task` +
`task=<route>` invokes a TypedLambdaFunction (@PreLoad) with Event Script-style
input[]/output[] data mapping. **STRATEGIC: this is intended to be the LAST built-in
skill** — composable functions become the extension mechanism for complex custom logic;
flows/subgraphs (graph.extension) remain for orchestration.

## Design decisions (Eric-approved)
1. `*` RHS = whole request body, Event Script sequential semantics (later field
   mappings merge into a `*`-seeded Map body; deep-copied to protect state machine).
2. Response headers exposed as `<node>.header` (like fetcher).
3. for_each[] + concurrency fork-join INCLUDED in v1 ("makes MiniGraph even more
   powerful than Event Script" — Eric).
4. No route allowlist (framework norm; same trust model as graph.extension).
5. Standalone skill (not a task:// scheme on graph.extension).

## Implementation notes (for future maintainers)
- Modeled on GraphExtension: Mono + po.eRequest single-call; po.request(batch) fork-join;
  setError/nextPath conventions; result/status/header node properties;
  performFetcherOutputMapping reused for output[] (RHS model./output. enforced there).
- Skill-local input RHS grammar in stageTaskParameter: `*` / `header.<k>` / body path
  (precedent: GraphApiFetcher.mapHttpParams). LHS via shared helper.getLhsOrConstant.
- `TASK` constant + `task` added to RESERVED_PARAMETERS in GraphLambdaFunction (blocks
  mapping overwrite + keeps it out of initializeWithNodeProperties).
- input/output/for_each auto-validated by GraphCommandService + CompileGraph
  MAPPING_PROPERTIES — zero wiring needed.
- Help file naming rule: `help/help graph-task.md` (dots→hyphens, literal space).
- Test graphs in test/resources/graph/unit-test-task-{1..5}.json registered in
  graphs.yaml → CompileGraph validates them at boot.
- Test functions v1.demo.task / v1.pojo.task under com.accenture.minigraph.tasks
  (component scan com.accenture picks up @PreLoad in test tree automatically).

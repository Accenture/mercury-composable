- **S106** (System.out) — `CompactAppender` + `JsonAppender` ARE the console log sinks (like log4j2's
  built-in ConsoleAppender); stdout is their purpose. → `@SuppressWarnings("java:S106")` on `append()`.
- **S112** (throws Exception) — `EntryPoint`, `StreamFunction`, `TypedLambdaFunction` (and `LambdaFunction`,

  throw anything and the engine catches it. Narrowing breaks all user code. → `@SuppressWarnings("java:S112")`.
- Alternatives noted for Eric: `// NOSONAR` (guaranteed honored, blunter) is the fallback if the field
  profile disables `@SuppressWarnings`; for the appenders a custom log4j2 **Layout** + built-in
  ConsoleAppender would remove app-code System.out entirely (heavier, behavior-equivalent).
- Only 2 product files use System.out (both these appenders); S112 `throws Exception` sources = the 4 core
  functional interfaces (implementations @Override and aren't re-flagged).

### The alert
`com.fasterxml.jackson.core:jackson-databind` >= 2.22.0, < 2.22.1 — per-property
`@JsonIgnoreProperties` exclusions bypassed when case-insensitive matching rebuilds the property map
from the unfiltered original (CWE-915, mass-assignment-style write). Flagged on
`helpers/kafka-standalone/pom.xml` (pins 2.22.0).

### Why it does not apply here (verified, not assumed)
The exploit needs BOTH preconditions; the repo has neither:
- Zero occurrences of `ACCEPT_CASE_INSENSITIVE_PROPERTIES` (as `MapperFeature` or per-property
  `@JsonFormat`) in Java sources.
- Zero occurrences of `@JsonIgnoreProperties` in Java sources.
- `helpers/kafka-standalone` never constructs an `ObjectMapper` at all — its databind pin only
  satisfies Kafka's transitive dependency.

### Why no version bump yet
Maven Central's latest jackson-databind is still **2.22.0** — the fixing 2.22.1 is unreleased
(alert shows "Patched version: None"; only the 3.x line has shipped its fix, 3.1.4 on 2026-06-04).
This is the same CVE already tracked as "no fix yet in the 2.x line" in the 2026-07-01 Snyk
dependency-update session.

### Recommendation given to Eric
Dismiss alert #28 as "vulnerable code is not in use"; bump when 2.22.1 lands (Dependabot will
re-flag on release — that's the reminder). The version is centralized: `jackson-2-bom.version`
property in ~30 poms + 3 explicit `jackson-databind` pins (platform-core, rest-spring-3,
kafka-standalone).

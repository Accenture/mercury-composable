### Finding: BDB is not virtual-thread-friendly (VT pinning) ‚Äî commit context
Validating vthread across the reactor (Eric asked to test (b) before flipping the default) revealed:
- vthread + **file**: platform-core 381, event-script 132, minimalist-kafka 79, **minigraph 55** ‚Äî all green.
- vthread + **bdb**: **minigraph 11 failures + 3 errors** (8s timeouts). Passes in isolation, breaks in the
  full suite ‚Üí a **concurrency/carrier-starvation** effect, not a per-test bug. Decisive A/B: same suite,
  vthread+file = green, vthread+bdb = broken ‚Üí **Berkeley DB's internal synchronized/latches PIN VT carriers**;
  under many per-route dispatch VTs contending on the shared BDB, the carrier pool starves.
- **Implication (Eric):** this likely explains BDB's poor high-pressure field behaviour better than GC did.
  Precise mechanisms: (1) *current* field runs loop dispatch ‚Üí BDB stalls the **event loop** (P0-measured);
  (2) VT pinning is *already* live in the field via the **`elastic.queue.cleanup` task** ‚Äî a plain function
  (only `@ZeroTracing`, NOT `@KernelThreadRunner`) that runs on a VT and does heavy `cleanLog()`/cursor BDB
  ops ‚Üí pins a carrier each time, and cleanup fires constantly under back-pressure. GC (P2: 2ms max pauses)
  is likely smaller than assumed. Net: BDB is architecturally at odds with a VT-first runtime; the file store
  (per-route, no shared lock, no synchronized) is the **only VT-compatible store** ‚Äî promotes it from
  "nicer/portable" to "required for off-loop dispatch".

### Simplification: one switch, two safe modes (commit `dd77f067`)
Collapsed the two orthogonal switches into one so vthread+bdb is **unreachable by construction** (better than
a runtime guard). `ElasticStore.supportsVirtualThreadDispatch()` ‚Äî the store declares its VT-safety
(`bdb`‚Üífalse, `file`‚Üítrue); `ServiceQueue` derives dispatch from it and the standalone `elastic.queue.dispatch`
config is **removed**. Single knob = `elastic.queue.store`: `bdb` (default) ‚áí inline loop dispatch (today's
safe behaviour); `file` ‚áí per-route virtual-thread dispatch off the loop. `DispatchBenchmark` mode label now
derives from the store. Verified: platform-core 381 green in both (no vthread log with bdb; vthread log fires
with file); minigraph 55 green with file.

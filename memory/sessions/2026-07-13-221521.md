## 1. Trace-propagation saga RESOLVED — gateway allow-list, no framework change

Eric's devops screen share found the root cause: the internal API gateway strips
`traceparent` AND `X-Trace-Id` (neither allow-listed); only `X-Correlation-Id` passes.
4.4.11 "worked" because `trace.http.header=X-Correlation-Id, X-Trace-Id` made the trace
ride the allow-listed first entry. My correction to the field write-up (verified in code
+ today's wire-level tests on v4.7.1): 4.6.3+ DOES emit X-Trace-Id alongside traceparent
— the gateway eats both. Eric REJECTED a proposed `legacy.trace.id` bridge flag: no
re-mixing of business cid and trace id — proper traceId/spanId/parentSpanId propagation
is the point. **Resolution: infra team to allow-list `traceparent` (+ `X-Trace-Id`) at
the gateway; zero app/framework change.** Open thread updated accordingly.

## 2. secondary.kafka.health (`feature/kafka-health-check` gap; branch
`feature/secondary-kafka-health`, commit `1d2023c2`)

Eric spotted that 4.8.3's `kafka.health` probes only the primary cluster — a twin-kafka
bridge with one dead cluster is broken even when the other is healthy. Added the twin:
`secondary.kafka.health` probing via the secondary consumer template. Implementation
per the module's reuse-seam conventions: KafkaHealthCheck's constructor seam widened to
**protected** + parameterized with a **service name** (dependency list shows `kafka` vs
`secondary.kafka`, each with its cluster's bootstrap as href) and a
`resolveDurationMs(key, fallbackKey, default)` helper implementing the twin fallback
convention (`secondary.kafka.health.timeout`/`.startup.grace` → `kafka.health.*` →
5s/30s). Same probe semantics (no-ACL Metadata request, startup grace placeholder, 503).
Bridge config: `mandatory.health.dependencies=kafka.health, secondary.kafka.health`
(Eric's proposal, term corrected from "mandatory.health.check"). 3 new tests incl.
probe-independence (dead secondary 503 while live instance healthy); twin-kafka 9 green,
minimalist-kafka 96 green. Docs: twin-kafka health section + config rows,
config-reference row, kafka-flow-adapter cross-ref. CHANGELOG deliberately untouched
(4.8.3 released; entry belongs to the next release bump).

### Root-caused the 582 ms `file` write outlier — it is NOT GC
Re-ran the worse-case `file` benchmark with `-Xlog:gc`. The stall reproduced (**582.6 ms write at
t=161.8s**), but the GC log shows the **max GC pause in the whole 180 s run was 2.0 ms, with zero pauses
> 100 ms** (healthy young collections, 333M→27M, no full GC). So the stall is an **OS dirty-page-flush
throttle**: writes only, late in the run (dirty pages accumulate), high-throughput unsync'd (we don't fsync).
**Decision (Eric):** no in-store background flusher (adds a thread, against the simplicity goal); instead
**elevate P5** (per-route VT off-loading — makes any on-loop spill stall harmless, the principled fix) and
**document tmpfs** as an immediate ops mitigation (transient store on RAM ⇒ no disk flush). Design spec §5.11,
P5 elevated to RECOMMENDED, P2 notes updated.

## What We Did

Diagnosed why AI agents (e.g. Kiro) struggle with the DSL — syntax taught by example, incomplete
hand-written context files, and no anti-drift link to code — and built a deterministic **DSL spec
layer** for MiniGraph, validated by a clean-context fresh agent. (Decision: `docs-dsl-spec`.)

- **Grounded the grammar in the shipped per-command help pages** (create/connect/instantiate/
  describe/inspect, the skill pages, data-dictionary, tutorial 3) — not from memory.
- **Wrote the spec kit** (committed `03a620fe` on `feature/docs-ai-human-rewrite`):
  - `command-reference.md` — rule-based grammar (lexical rules, namespaces, every command, node
    types, skill→property matrix, invariants).
  - `minigraph-commands.json` — machine-readable catalog (20 commands, 7 skills, 11 namespaces).
  - `ai-agent-guide.md` — companion-endpoint contract + pre-send checklist + build recipe.
  - `scripts/check-minigraph-grammar.py` + `.github/workflows/docs.yml` — a CI **drift test**
    (spec ↔ shipped help ↔ engine routes) + strict docs build. Verified the test catches injected
    drift and that routes are declared as `static final String ROUTE` constants.
- **Fresh-agent validation (the proof):** spawned a clean-context agent with ONLY the 5 spec docs
  and a build task. **Pass 1** built the happy path but flagged 9 gaps — 3 real: config nodes
  needing connections (it guessed `connect ... with uses`), the `response.*` namespace, and
  node-type casing. Fixed all three (tutorial 3 confirms config nodes need NO connections). **Pass 2**
  answered all three correctly and confidently and did not guess; it caught one residual
  contradiction (invariant #3 still said "names and types lowercase") — fixed, plus 2 minor
  clarifications (URL `{id}`↔`path_parameter.id`; the bare `input[]` declaration form).
- **Closed Open Thread `thread-ci-docs-build`** — the new workflow is the docs' first CI gate.

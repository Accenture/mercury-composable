Per Eric's hint ("start the MainApp in @BeforeAll, publish + consume"):
- **`MainAppTest`** (new, `example` pkg): `@BeforeAll new MainApp().start([])` with **`dual.servers=true`**
  in the test `application.properties` → drives the *two-broker* branch of `MainApp.start` (boots 9092 +

  up) and both `EmbeddedKafka` constructor variants; then a real producer/consumer round-trips 3 messages
  on 9092. Calling `start()` directly (not `AutoStart`/`MainApp.main`) avoids the full-platform boot and the
  `web.component.scan` @MainApplication-discovery dependency (`AppStarter` needs it; the minimal test
  config lacks it).
- **`EmbeddedKafkaTest`** (rewritten, `system` pkg — same pkg reaches package-private/private seams): no
  broker; covers the fail-fast/cleanup branches deterministically — shutdown-before-start no-op, missing
  properties resource (reflection sets private `serverPropPath`), absent `log.dirs`, and re-formatting an
  existing dir (reflection invokes private `formatStorage(Properties)` → runs the real KRaft `Formatter`).
- Deleted the old `EmbeddedKafkaTest` boot (direct `startServer()` on 9092) — a 2nd 9092 boot in the same
  surefire JVM would clash with the MainApp boot; the MainApp path covers `startServer`/`formatStorage`
  happy paths for both brokers anyway (and `run()`, which the old direct-`startServer` test skipped).

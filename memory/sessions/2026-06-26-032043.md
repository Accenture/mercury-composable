**What shipped:**

- **New system library `system/minimalist-kafka`** (`org.platformlambda.mini.kafka`, depends on
  `event-script-engine`). The reusable Kafka pair that production installs otherwise hand-roll:
  - `SimpleKafkaNotification` (`simple.kafka.notification`) — drop-n-forget egress; stamps a fresh
    W3C `traceparent` from its own span (built from `new PostOffice(headers, instance)`).
  - `KafkaFlowAdapter` / `KafkaFlowConsumer` — one poll-loop thread per topic; routes each message
    into an Event Script flow via the **low-level `PostOffice` API** (not `FlowExecutor`, which
    doesn't expose the inbound span), chaining the parent span with `forward.setSpanId(parentSpanId)`;
    **commit-after-process** with `max.poll.records=1` + manual commit = at-least-once.
  - `KafkaFlowAutoStart` (`@MainApplication`, runs after the engine so flow routing works) +
    `KafkaRuntime` singletons. Auto-registers from the jar via base-package component scan.
  - Standalone e2e (`KafkaFlowAdapterTest`) on an embedded KRaft broker: notification → topic →
    adapter → sink flow, asserting trace-id continuity. **4 tests, 87% line/instruction coverage.**

- **`sync-over-async` refactored onto it.** Deleted the moved Kafka classes; the pom now depends on
  `minimalist-kafka` (dropped the direct `event-script-engine` + `kafka-clients` deps — both
  transitive now). The module is now **purely the Redis cross-pod return-route engine** (`sync` pkg)
  + config. The Phase-3 REST facade (`test.endpoint`) and the full composable round-trip live in the
  tests. **20 tests, 96% coverage**; `RestFlowMvpTest` still proves REST → Kafka → SoR → Kafka →
  reply → Redis → 200/408 end-to-end through the dependency.

- **OTel span propagation across Kafka** (the "loose end" Eric flagged): no `event-script-engine`
  change needed — the wiring already exists (`WorkerHandler:103` uses the incoming event's span-id as
  the function's parent). Egress stamps its own span into the Kafka `traceparent`; ingress parses it
  and sets it as the flow's parent span. Result: one continuous trace from REST through every Kafka
  hop (validated against Eric's `telemetry.txt`: 10 spans, bridge spans = the two notification hops).

- **Reactor:** added `system/minimalist-kafka` after `mini-scheduler`; builds green in dependency
  order on JDK 21 (minimalist-kafka → sync-over-async).

## What shipped
- **Per-type serde strategy** (commit `1bc2731e`). Refactored `SchemaCodec` from inline JSON logic into a
  `SchemaSerde` interface + `JsonSchemaSerde` (existing logic moved verbatim) + `AvroSchemaSerde`. The codec
  now just dispatches — by the `schema-type` header on produce, by the registered type (looked up from the
  embedded id) on consume. The producer (`simple.kafka.notification`) and consumer (`KafkaFlowConsumer`)
  were already id/type-generic, so **Protobuf is now a one-class drop-in**.
- **`AvroConversions`** — Map ⇄ GenericRecord. Serialize **walks the writer schema, not the value**, so an
  absent field takes its **schema default**; a partial input serializes cleanly where Avro's own strict JSON
  decoder would reject it. Handles records, `[null,X]` unions, arrays, maps, enums, numeric coercion, bytes.
- `FileCachedSchemaRegistryClient` reconstructs Avro from the on-disk cache too; pom adds
  `io.confluent:kafka-avro-serializer:8.2.0` (transitive kafka-clients excluded).
- **Demo Avro path** (commit `6fc8c56c`): `avro-topic-1/2`, `/api/sync-to-async-avro`, three `*-avro.yml`
  flows, `SystemOfRecordAvro` (FLAT reply — Avro records are closed-shape) + `SoaReplyAvro`, adapter
  bindings, seed `registry/schemas.json` id 2 (Avro SyncDemoMessage, all fields default `""`), node topics.

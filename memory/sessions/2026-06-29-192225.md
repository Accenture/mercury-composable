## What shipped
- **ProtobufSchemaSerde + ProtobufConversions** (commit `bdab28f3`). The serde-strategy refactor from the
  Avro phase made this a drop-in: register a third `SchemaSerde` in `SchemaCodec.fromConfig` (+
  `ProtobufSchemaProvider` on the client); producer/consumer unchanged.
  - `ProtobufConversions`: Map ⇄ `DynamicMessage` built reflectively against the registered schema's
    descriptor — **no generated .proto classes**. `toMessage` walks the descriptor and skips fields absent
    from the Map, so proto3's implicit default applies on read (the "partial input serializes cleanly"
    behavior, free in proto3 vs Avro's explicit defaults). Handles scalars (numeric coercion), enums, nested
    messages, repeated.
  - `FileCachedSchemaRegistryClient` reconstructs Protobuf from cache; pom adds
    `io.confluent:kafka-protobuf-serializer:8.2.0`.
- **Demo Protobuf path** (commit `bc9be976`): `protobuf-topic-1/2`, `/api/sync-to-async-protobuf`, three
  `*-protobuf.yml` flows, `SystemOfRecordProtobuf` (FLAT reply — protobuf is closed-shape like Avro) +
  `SoaReplyProtobuf`, adapter bindings, seed `registry/schemas.json` id 3 (proto3 SyncDemoMessage), node
  topics, README section + a closing "three wire formats" summary.

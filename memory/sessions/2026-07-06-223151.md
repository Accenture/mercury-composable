exhaustively — they only touch a handful of column types.

Added **`SqlStatementTypesTest`** (pure unit, no DB) in `com.accenture.postgres.tests`:
- value data-type detection for every supported type (bind + assert `getClassMapping`);
- string decode/restore per type — done **directly** via `setNamedParams` + the live mutable
  `getClassMapping().put(...)` with controlled strings, NOT a SimpleMapper JSON round-trip. Reason: the
  mapper serializes temporal types with a **space** separator (`2026-01-02 03:04:05`) but the restore
  path calls `LocalDateTime.parse`/`OffsetDateTime.parse` which need ISO `T` — so a JSON round-trip
  throws `DateTimeParseException` for local-datetime/offset-datetime (a latent format mismatch, left
  as-is);
- null-type → JDBC class / `java.sql.Types` mapping for all types; fail-fast paths (unsupported value,
  unsupported/`Character` null class, unbound null name, malformed list bindings, re-bind after bound).
- `SqlPreparedStatement` 68% → **94.2%**.

Also added `pgRequestValidationGuards` to `ReactiveDbTest` (reuses the booted app) covering `PgRequest`
